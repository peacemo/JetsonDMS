# Jetson-Based Driver Status Monitoring System

## I. Project Technology Stack Overview
| Module       | Technical Direction      | Core Content                                         |
| :----------- | :----------------------- | :--------------------------------------------------- |
| Vision Module| Embedded Vision          | Camera capture, Image preprocessing, Face detection, Eye state analysis |
| Communication Module | In-vehicle Communication | CAN driver, SocketCAN, UDS protocol stack, Multi-frame transmission |
| Decision Module | State Machine + Diagnosis| Fatigue algorithm, State management, Diagnostic service interface |
| System Layer | Embedded Linux           | Multi-threaded architecture, Inter-module communication, Resource management, Makefile build |

## II. Project Execution Three Phases (Weekly Granularity)

### **Phase 1: Foundation Environment & Minimal System (1 Week)**

**Goal:** Verify hardware functionality and ensure each module works independently.

*   **Vision Subtasks**
    *   Jetson system flashing and camera driver verification.
    *   Finalize image acquisition program (continuous image capture).
    *   Determine the source of the face detection model.
*   **Communication Subtasks**
    *   Load CAN driver module and configure interfaces.
    *   Verify connection between the Guangcheng analyzer and the PC.
    *   Test CAN send/receive on Jetson (loopback mode).
*   **System Subtasks**
    *   Confirm the cross-compilation environment.
    *   Establish the project directory structure.

**Phase Milestone:** Jetson can send and receive CAN messages in loopback mode, and the camera can output continuous images.

### **Phase 2: Independent Implementation of Three Modules (2-3 Weeks)**

**Goal:** Each module completes its core functionality and can be tested independently.

*   **Vision Module**
    *   Port the face detection algorithm to Jetson.
    *   Implement eye keypoint localization.
    *   Test and record model inference performance.
*   **Communication Module**
    *   Encapsulate the SocketCAN C language interface.
    *   Implement ISO 15765-2 multi-frame transmission logic.
    *   Build the basic UDS service framework (0x10/0x22/0x2E).
*   **Decision Module**
    *   Design the fatigue judgment algorithm (e.g., PERCLOS).
    *   Design the state machine: Normal / Fatigued / Severely Fatigued.
    *   Define the custom DID (0xF190) data structure.

**Phase Milestone:** Vision module can output face bounding boxes + eye state; Communication module can send/receive single-frame UDS messages; Decision module has clear input/output definitions.

### **Phase 3: System Integration & Closed-Loop (1-2 Weeks)**

**Goal:** Integrate and debug the three modules, creating a complete demonstration use case.

*   **Integration Tasks**
    *   Design multi-threaded architecture: Vision Thread + CAN Thread + Decision Thread.
    *   Design inter-thread communication mechanism (shared memory / message queue).
    *   Build the main loop framework.
*   **Diagnostic Closed-Loop**
    *   Write decision results to a DID via UDS.
    *   Read and verify the DID on the PC using the Guangcheng analyzer.
    *   (Optional) Simulate multi-frame flashing.
*   **Testing & Optimization**
    *   Measure end-to-end latency.
    *   Analyze memory usage.
    *   Handle exception cases (CAN disconnection, camera loss).

**Phase Milestone:** Vision detects eye closure -> Jetson sends UDS message over CAN bus -> PC receives fatigue status.
