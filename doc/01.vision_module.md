# Vision Module Design 

This document reflects the current implementation in the `src/vision` module.

## Functionality

The implemented pipeline performs:

1. Camera initialization and frame capture
2. Image preprocessing (color conversion, resize, normalization)
3. TensorRT inference
4. Status/confidence reporting through C API result structs

## Software Stack

1. OpenCV C++ (`VideoCapture`, color conversion, resize, image write)
2. TensorRT C++ runtime API
3. CUDA runtime API (`cudaMalloc`, `cudaMemcpyAsync`, stream sync)

## Public C API

Header: `src/vision/include/vision_api.h`

- `int vision_init(int camera_id, const char* model_path)`
	- Returns `0` on success, `-1` on failure.
- `int vision_detect(DetectResult* result)`
	- Returns `0` on success, `-1` on failure.
- `void vision_cleanup(void)`

Data types are defined in `src/vision/include/vision_types.h`:

- `DetectStatus`
	- `DETECT_STATUS_UNKNOWN = 0`
	- `DETECT_STATUS_NORMAL = 1`
	- `DETECT_STATUS_DROWSY = 2`
	- `DETECT_STATUS_DISTRACTED = 3`
	- `DETECT_STATUS_ERROR = -1`
- `DetectResult`
	- `DetectStatus status`
	- `float confidence`
	- `int frame_id`

## Internal Module Design

### 1. Camera Module

Files:

- `src/vision/include/internal/camera.hpp`
- `src/vision/camera.cpp`

Implemented API:

- `bool Camera::init(int camera_id)`
- `bool Camera::isReady() const`
- `cv::Mat Camera::captureFrame()`
- `void Camera::release()`

Behavior:

- Uses fixed default capture settings: `1280x720 @ 30 FPS`.
- Camera open strategy:
	1. Try Jetson CSI pipeline via `cv::CAP_GSTREAMER`.
	2. If CSI open fails, fallback to `cap_.open(camera_id)` device path.
- CSI pipeline includes low-latency appsink settings:
	- `max-buffers=1 drop=true sync=false`
- CSI warmup reads 3 frames after open.
- Frame capture retries up to 5 times, with 30 ms delay between retries.
- `captureFrame()` returns empty `cv::Mat` on failure.

Pipeline template:

`nvarguscamerasrc sensor-id=<id> ! video/x-raw(memory:NVMM), width=1280, height=720, framerate=30/1 ! nvvidconv ! video/x-raw, format=BGRx ! videoconvert ! video/x-raw, format=BGR ! appsink max-buffers=1 drop=true sync=false`

### 2. Preprocessing Module

Files:

- `src/vision/include/internal/preprocessing.hpp`
- `src/vision/preprocessing.cpp`

Implemented API:

- `bool Preprocessor::init(int width, int height)`
- `bool Preprocessor::process(const cv::Mat& input, cv::Mat& output)`

Behavior:

- Supports input channels: 1, 3, or 4.
- Converts input to RGB:
	- BGR -> RGB
	- BGRA -> RGB
	- GRAY -> RGB
- Resizes to `224x224` using `cv::INTER_LINEAR`.
- Normalizes to `CV_32FC3` with scale `1.0 / 255.0`.

Note:

- `init(width, height)` stores target dimensions, but current resize logic is hard-coded to `224x224`.

### 3. TensorRT Inference Module

Files:

- `src/vision/include/internal/trt_model.hpp`
- `src/vision/trt_model.cpp`

Implemented API:

- `bool TrtModel::init(const std::string& model_path)`
- `bool TrtModel::infer(const cv::Mat& image, DetectResult& result)`
- `void TrtModel::release()`

Behavior:

- Loads a serialized TensorRT engine from file.
- Uses TensorRT name-based I/O API (`getNbIOTensors`, `setInputShape`, `setTensorAddress`, `enqueueV3`).
- Supports input/output tensor data types:
	- `kFLOAT`
	- `kHALF`
- Supports static and dynamic input shapes.
- Converts OpenCV HWC image to CHW layout before GPU copy.
- Runs inference on CUDA stream and reads back output asynchronously.
- Sets `result.frame_id` using an internal frame counter.

Output parsing (`processOutput`):

- Assumes binary classifier with at least 2 output values.
- Class order is defined as:
	- index 0: drowsy
	- index 1: non-drowsy
- If outputs already look like probabilities, uses them directly.
- Otherwise applies softmax.
- Final status mapping:
	- `prob_drowsy >= prob_non_drowsy` -> `DETECT_STATUS_DROWSY`
	- else -> `DETECT_STATUS_NORMAL`

### 4. Vision Pipeline Orchestration

Files:

- `src/vision/include/internal/vision_impl.hpp`
- `src/vision/vision_impl.cpp`

Class: `vision::VisionPipeline`

Implemented flow:

- `init(camera_id, model_path)`:
	1. Initialize camera.
	2. Capture one initial frame.
	3. Save initial frame to `output.jpg` in current working directory.
	4. Initialize preprocessor with `(224, 224)`.
	5. Initialize TensorRT model.
- `detect(result)`:
	1. Capture frame.
	2. Preprocess frame.
	3. Run inference.
	4. Fill `DetectResult`.
- `cleanup()`:
	- Releases inference and camera resources.

## C API Wrapper Behavior

File: `src/vision/vision_api.cpp`

- Uses global singleton-like pipeline instance:
	- `static std::unique_ptr<vision::VisionPipeline> g_pipeline`
- `vision_init`:
	- Fails if already initialized.
	- Validates `model_path != nullptr`.
- `vision_detect`:
	- Fails if pipeline is not initialized.
	- Validates `result != nullptr`.
	- On failure sets error fields when possible.
- `vision_cleanup`:
	- Safe to call repeatedly.

## Current Prototype Notes

1. Detection is currently binary (drowsy vs normal).
2. `DETECT_STATUS_DISTRACTED` exists in enum but is not produced by current inference logic.
3. Initialization currently writes one captured frame to `output.jpg` as an artifact.
4. Preprocessor target size parameters are not fully wired (hard-coded resize to `224x224`).
